Voy a implementar la seguridad RLS (Row Level Security) mediante una migración de Prisma. Esto dejará registrado en tu código la configuración de seguridad de Supabase.

### Pasos Técnicos

1.  **Crear la migración**: Generaré un archivo SQL vacío con Prisma (`--create-only`) llamado `add_rls_policies`.
2.  **Escribir las Políticas (SQL)**:
    *   **Función `is_admin()`**: Una función segura para verificar si el usuario actual tiene el rol 'admin'.
    *   **Habilitar RLS**: Activar seguridad en `Users`, `Role`, `Permission`, `RoleAuditLog`.
    *   **Políticas de Lectura**:
        *   `Users`: Cada quien ve lo suyo, Admin ve todo.
        *   `Role/Permission`: Visibles para todos los usuarios logueados (necesario para la UI).
        *   `RoleAuditLog`: Solo Admin.
    *   **Políticas de Escritura**:
        *   `Users`: Usuarios editan su perfil, Admin edita cualquiera.
        *   `Role/Permission`: Solo Admin puede modificar.
3.  **Aplicar la migración**: Ejecutar la migración para impactar los cambios en la base de datos.
