# Plan de Diseño de Interfaz: Security & Admin Console

He analizado tu aplicación y detectado que, aunque el código es robusto (Next.js 16, Prisma, RBAC), el diseño visual utiliza los valores por defecto de shadcn/ui ("Indigo Theme"), lo que le da una apariencia genérica.

Para aplicar la skill `interface-design` correctamente, propongo transformar la identidad visual para que coincida con la funcionalidad de **Gestión de Roles y Seguridad**.

## 1. Definición del Dominio
*   **Tipo de Producto:** Infraestructura de Administración / Panel de Seguridad.
*   **Conceptos Clave:** Control, Auditoría, Jerarquía, Permisos, Logs.
*   **Objetivo:** Transmitir confianza, precisión y estabilidad técnica.

## 2. Propuesta de Diseño (Mandate)

### Mundo de Color: "Technical Slate"
En lugar del azul genérico, usaremos una base de **Slate** (gris con tinte azulado/frío) que es estándar en herramientas de desarrollo de alta calidad, con acentos en **Violeta** (tecnológico) o **Monocromático**.

*   **Fondo:** Oscuro profundo, no negro puro (ej. Slate 950).
*   **Superficies:** Capas sutiles definidas por bordes, no por sombras difusas.
*   **Bordes:** Visibles, técnicos, de alto contraste en modo oscuro.

### Firma Visual: "Micro-Borders & Density"
*   **Radio de Borde:** Reducir de `0.5rem` (amigable) a `0.25rem` o `0.3rem` (técnico/preciso).
*   **Tipografía:** Inter (sans) para UI, pero introduciremos una fuente **Monospace** (JetBrains Mono o Geist Mono) para IDs, fechas, logs y datos técnicos.
*   **Densidad:** Aumentar la densidad de información en tablas y listas.

### Rechazo de Defaults
*   ❌ **Rechazar:** Sombras grandes y suaves (`shadow-lg`).
    *   ✅ **Usar:** Bordes definidos y `shadow-sm` o planos.
*   ❌ **Rechazar:** Colores de estado brillantes (Rojo/Verde puro).
    *   ✅ **Usar:** Colores semánticos desaturados y con fondo sutil (ej. texto rojo sobre fondo rojo muy suave).

## 3. Pasos de Implementación

1.  **Crear Sistema de Diseño:** Generar el archivo `.interface-design/system.md` para documentar estas decisiones y guiar futuros desarrollos.
2.  **Refinar Variables CSS:** Modificar `app/globals.css` para cambiar la paleta base a **Slate** y ajustar el radio de borde (`--radius`) para que sea más "sharp".
3.  **Ajustar Tipografía:** Asegurar que la fuente monospace esté disponible para datos técnicos.
